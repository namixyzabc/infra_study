# バックアップの3つの方式を解説します

## 📦 フルバックアップ
**すべてのデータを丸ごとバックアップする方式**

### 特徴
- 毎回、全データをコピー
- 復元が最も簡単（最新のバックアップだけでOK）
- 時間とストレージ容量が最もかかる

### イメージ
```
月曜: 全データ（A,B,C,D,E）
火曜: 全データ（A,B,C,D,E）
水曜: 全データ（A,B,C,D,E）
```

---

## 📊 差分バックアップ
**最後のフルバックアップ以降の変更分を保存**

### 特徴
- 基準は常に「最後のフルバックアップ」
- 復元には「フルバックアップ + 最新の差分」の2つが必要
- 時間経過とともにバックアップサイズが増える

### イメージ
```
日曜: 全データ（A,B,C,D,E）← フル
月曜: B'（Bの変更分）
火曜: B',C'（日曜以降の変更分すべて）
水曜: B',C',D'（日曜以降の変更分すべて）
```

---

## 🔄 増分バックアップ
**前回のバックアップ以降の変更分だけを保存**

### 特徴
- 基準は「直前のバックアップ」
- 最も容量と時間を節約できる
- 復元には「フルバックアップ + すべての増分」が必要

### イメージ
```
日曜: 全データ（A,B,C,D,E）← フル
月曜: B'（日曜以降の変更）
火曜: C'（月曜以降の変更）
水曜: D'（火曜以降の変更）
```

---

## 📋 比較表

| 項目 | フル | 差分 | 増分 |
|------|------|------|------|
| バックアップ時間 | ⏱️⏱️⏱️ | ⏱️⏱️ | ⏱️ |
| 必要容量 | 💾💾💾 | 💾💾 | 💾 |
| 復元の簡単さ | ⭐⭐⭐ | ⭐⭐ | ⭐ |
| 復元に必要なファイル数 | 1個 | 2個 | 複数個 |

## 💡 実際の運用例

多くの企業では**組み合わせて使用**します：

```
日曜: フルバックアップ
月曜: 増分バックアップ
火曜: 増分バックアップ
水曜: 増分バックアップ
木曜: 増分バックアップ
金曜: 増分バックアップ
土曜: 差分バックアップ
```

この方法で、効率性と安全性のバランスを取ります！
