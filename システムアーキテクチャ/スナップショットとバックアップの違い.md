# スナップショットとバックアップの違い

## 基本的な定義

### スナップショット
- 特定時点のシステム状態を「記録」したもの
- 元のデータと同じストレージ上に保存
- 差分データのみを保持する仕組みが一般的

### バックアップ
- データを「複製」したもの
- 別の場所・別のストレージに保存
- データ全体を複製するのが基本

## 主な違い

| 項目 | スナップショット | バックアップ |
|------|-----------------|--------------|
| **目的** | 迅速な復元・一時的な保護 | 長期保存・災害対策 |
| **保存場所** | 同一ストレージ | 別のストレージ・外部媒体 |
| **作成速度** | 高速（秒～数分） | 低速（分～数時間） |
| **容量** | 少ない（差分のみ） | 多い（全データ複製） |
| **復元速度** | 高速 | 低速 |
| **保存期間** | 短期間 | 長期間 |
| **災害対策** | 不適切 | 適切 |

## 利用シーンの例

### スナップショットが適している場合
- システムアップデート前の一時保存
- 開発環境での試験前のバックアップ
- 短期間の変更履歴管理
- 迅速なロールバックが必要な場合

### バックアップが適している場合
- 定期的なデータ保護
- ハードウェア障害への対策
- ランサムウェア対策
- 法的要件による長期保存
- 災害復旧（DR）対策

## 重要なポイント

### スナップショットの注意点
- 元のストレージが故障すると使用不可
- 長期保存には向かない
- スナップショット数が増えるとパフォーマンス低下

### バックアップの注意点
- 作成に時間がかかる
- ストレージ容量を多く消費
- 世代管理が必要

## 推奨される運用

**両方を組み合わせて使用するのがベストプラクティス**

- **短期的な保護** → スナップショット（時間単位・日次）
- **長期的な保護** → バックアップ（週次・月次）
- **災害対策** → オフサイトバックアップ（遠隔地保存）
