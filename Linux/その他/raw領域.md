# raw領域のバックアップについて

## raw領域とは

**raw領域（ローデバイス）** とは、ファイルシステムを経由せずに、ディスクやパーティションに直接アクセスする方式です。

```
通常のアクセス方法：
アプリケーション → ファイルシステム → ディスク

raw領域アクセス：
アプリケーション → ディスク（直接）
```

## ファイルシステムをスキップするとは

### 通常のバックアップ
- ファイルやディレクトリ単位でコピー
- OSのファイルシステム（NTFS、ext4など）を介してアクセス
- ファイル名、権限、タイムスタンプなどのメタデータを処理

### raw領域のバックアップ
- **ブロック単位**でディスク全体をそのままコピー
- ファイルシステムの構造を無視
- ディスクの物理的な内容をそのまま読み書き

## 具体例

```bash
# Linuxでのraw領域バックアップ例
dd if=/dev/sda of=/backup/disk.img bs=4M

# 説明：
# if=/dev/sda  ：ディスクデバイスから直接読み込み
# of=disk.img  ：イメージファイルに書き込み
# bs=4M        ：4MBブロック単位で処理
```

## メリット

1. **完全なコピー**
   - パーティション情報、ブートセクター含めすべて保存

2. **ファイルシステム非依存**
   - どんなファイルシステムでも同じ方法でバックアップ可能

3. **データベース向き**
   - Oracleなどのデータベースが好んで使用

## デメリット

1. **容量が大きい**
   - 使用していない領域も含めてすべてバックアップ

2. **時間がかかる**
   - ディスク全体を読み込むため

3. **柔軟性が低い**
   - 個別ファイルの復元が困難

## 主な用途

- データベースのバックアップ
- ディスク全体のクローン作成
- システム全体の完全バックアップ
- 災害復旧（DR）対策
