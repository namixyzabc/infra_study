# Azureのリソースグループ

## 📦 基本概念

**リソースグループ**は、Azure上で関連するリソース（サービスや機能）をまとめて管理するための「**入れ物（コンテナ）**」です。

### 身近な例え
- **フォルダ**のようなもの：パソコンで関連ファイルをフォルダにまとめるのと同じ
- **引き出し**のようなもの：プロジェクトごとに物を分けて収納する感覚

---

## 🎯 何が入るの？

リソースグループには、以下のようなAzureリソースを格納します：

- 仮想マシン（VM）
- データベース
- ストレージアカウント
- 仮想ネットワーク
- Webアプリ
- など

---

## ✨ 主なメリット

### 1. **まとめて管理**
```
リソースグループ「本番環境」
├─ Webサーバー
├─ データベース
└─ ストレージ
```
関連リソースを1箇所で確認・管理できます

### 2. **まとめて削除**
プロジェクト終了時、リソースグループごと削除すれば、中のリソース全てが一括削除されます

### 3. **権限管理**
リソースグループ単位でアクセス権限を設定できます

### 4. **コスト管理**
リソースグループ別に利用料金を確認できます

---

## 📝 重要なルール

| ルール | 説明 |
|--------|------|
| **必須** | すべてのリソースは必ずどこかのリソースグループに属する |
| **1つだけ** | 1つのリソースは1つのリソースグループにのみ所属 |
| **地域指定** | リソースグループ自体に地域（リージョン）を指定する |
| **移動可能** | 後から別のリソースグループに移動できる（一部例外あり） |

## 💡 実践的な使い方

### 分け方の例

**環境別**

- リソースグループ「本番環境」

**プロジェクト別**
- リソースグループ「ECサイトプロジェクト」
- リソースグループ「分析システムプロジェクト」

**部門別**
- リソースグループ「営業部」
- リソースグループ「開発部」

---

## 🚀 作成手順（簡単）

1. Azureポータルにログイン
2. 「リソースグループ」を検索
3. 「作成」をクリック
4. 名前と地域を指定
5. 「確認および作成」

---

## ⚠️ よくある質問

**Q: リソースグループを削除するとどうなる？**
A: 中のリソースも全て削除されます（注意！）

**Q: 費用はかかる？**
A: リソースグループ自体は無料です

**Q: 後から名前変更できる？**
A: できません。最初に適切な命名を！
