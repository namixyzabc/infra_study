# Azureのテナントとサブスクリプションについて


## 🏢 テナント (Tenant)

### 概要
- **会社や組織そのもの**を表す最上位の管理単位
- Azure Active Directory (Azure AD) の1つのインスタンス
- 組織全体のユーザー、グループ、認証を管理

### 特徴
```
テナント = 会社のビル全体
├─ ユーザー管理（社員名簿）
├─ 認証・セキュリティ（入館証）
└─ 組織全体の設定
```

- テナントIDは固有（例：`contoso.onmicrosoft.com`）
- 1つの組織は通常1つのテナント
- すべてのAzureリソースはテナントに紐づく

## 💳 サブスクリプション (Subscription)

### 概要
- **課金の単位**であり、**リソースの入れ物**
- 実際にAzureサービスを使うための契約

### 特徴
```
サブスクリプション = 部署ごとの予算枠
├─ 仮想マシン
├─ データベース
├─ ストレージ
└─ その他のAzureリソース
```

- 1つのテナント内に複数のサブスクリプションを持てる
- 部署別、プロジェクト別、環境別（開発/本番）に分けられる

## 🔗 関係性

```
テナント（組織全体）
 └─ サブスクリプションA（営業部の予算）
      ├─ VM1
      └─ DB1
 └─ サブスクリプションB（開発部の予算）
      ├─ VM2
      └─ Storage1
```

## 💡 具体例

| 要素 | 例え | 実際の使い方 |
|------|------|--------------|
| **テナント** | ABC株式会社 | `abc.onmicrosoft.com` |
| **サブスクリプション1** | 営業部の予算 | 営業システムのリソース |
| **サブスクリプション2** | 開発部の予算 | 開発環境のリソース |

## ✅ まとめ

- **テナント**：組織の枠組み（誰がアクセスできるか）
- **サブスクリプション**：お金の枠組み（どう課金するか）

この2つの階層構造で、Azureは大規模な組織でも効率的に管理できるようになっています。
