# Azureのストレージアカウント

## 📦 ストレージアカウントとは

Azureでデータを保存するための**基本的な入れ物（コンテナ）**です。クラウド上の倉庫のようなもので、様々な種類のデータを保存できます。

## 🗂️ 保存できるデータの種類

ストレージアカウント内では、以下の4つのサービスが利用できます:

### 1. **Blob Storage（ブロブストレージ）**
- 画像、動画、ログファイルなど、あらゆる種類のファイルを保存
- 最も一般的に使われる

### 2. **File Storage（ファイルストレージ）**
- 従来のファイル共有（ネットワークドライブ）のようなもの
- 複数のサーバーから同時アクセス可能

### 3. **Queue Storage（キューストレージ）**
- メッセージを一時的に保存
- アプリケーション間の連携に使用

### 4. **Table Storage（テーブルストレージ）**
- NoSQLデータベースのような構造化データを保存

## 💡 主な特徴

| 特徴 | 説明 |
|------|------|
| **高い可用性** | データは自動的に複数箇所にコピーされる |
| **スケーラビリティ** | 容量を気にせず使える |
| **セキュリティ** | 暗号化やアクセス制御が可能 |
| **低コスト** | 使った分だけ課金 |

## 🎯 よくある使用例

- **Webサイト**: 画像や動画の配信
- **バックアップ**: システムのバックアップデータ保存
- **ログ保存**: アプリケーションのログファイル
- **データ分析**: 大量データの保存と分析

## 🔑 作成時の主な設定項目

1. **パフォーマンス**: Standard（標準）/ Premium（高速）
2. **レプリケーション**: データのコピー方法（冗長性）
3. **アクセス層**: Hot（頻繁アクセス）/ Cool（低頻度）/ Archive（アーカイブ）
